## 1. What is Reconnaissance

Reconnaissance is the process of gathering information about a system, network, or organization before security testing.

In cyber security, reconnaissance is often the first step to understand how a system works.

Both attackers and security professionals use reconnaissance to learn about their target.

---

## 2. Why Information Comes First

Good security testing starts with understanding the system.

Before testing, professionals often want to know:
- what systems exist
- what technologies are used
- how services are exposed to the internet
- how users interact with the system

Gathering this information helps identify possible risk areas.

---

## 3. Passive vs Active Reconnaissance

There are two general approaches.

Passive reconnaissance collects information without directly interacting with the target system.

Examples include:
- public websites
- documentation
- public records
- open-source information

Active reconnaissance involves direct interaction with systems to gather information.

Security professionals must always have permission before active reconnaissance.

---

## 4. What Attackers Look For

Attackers often search for information such as:
- open network services
- exposed web applications
- software technologies used by a website
- public information about systems

They look for weaknesses that could be exploited later.

---

## 5. Why Security Professionals Use Recon

Ethical security professionals also use reconnaissance.

Their goal is to find weaknesses before attackers do.

By understanding system structure, security teams can:
- identify risks
- improve defenses
- strengthen system configurations

---

## 6. Key Takeaways

- Reconnaissance is the process of gathering information about systems.
- It is often the first step in security testing.
- Passive reconnaissance collects publicly available information.
- Active reconnaissance interacts directly with systems.
- Security professionals use reconnaissance to identify weaknesses and improve security.
