## Introduction

Monitoring systems generate alerts when something unusual happens.

Alerts help security teams detect possible threats.

However, not every alert means an attack is actually happening.

Because of this, analysts must carefully review alerts before taking action.

## What an Alert Is

An alert is a notification generated by a security system when suspicious activity is detected.

Security systems may create alerts for events such as:
- multiple failed login attempts
- unusual network traffic
- access to sensitive files
- suspicious software behavior

Alerts help security teams notice potential threats quickly.

## What a False Positive Is

A false positive happens when a security system generates an alert for activity that is actually normal.

For example, a system might flag a login from another city as suspicious, but it could simply be an employee traveling.

False positives are common in security monitoring.

## True Positives vs False Positives

True Positive:
- an alert that correctly identifies a real security threat

False Positive:
- an alert triggered by normal activity

Security analysts must review alerts to determine which type they are dealing with.

## Why Alert Investigation Matters

If analysts ignore alerts, real attacks could go undetected.

But if they treat every alert as a serious incident, security teams could become overwhelmed.

Blue Team analysts must analyze alerts carefully and prioritize real threats.

## Example Scenario

A monitoring system generates an alert because an employee logged in from a new device.

The analyst investigates and discovers the employee recently received a new company laptop.

This alert is a false positive.

## Key Takeaways

- Alerts notify security teams about suspicious activity.
- Not every alert indicates a real attack.
- False positives occur when normal activity triggers an alert.
- Security analysts must review alerts carefully.
