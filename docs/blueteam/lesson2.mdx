## Introduction

No single security control can fully protect a system.

Attackers may eventually bypass one defense.

Because of this, cybersecurity uses a concept called Defense-in-Depth.

Defense-in-Depth means using multiple layers of security to protect systems and data.

If one layer fails, other layers can still stop the attack.

## Why Multiple Layers Matter

Imagine protecting a building.

Instead of only having a lock on the door, you may also have:
- security cameras
- alarm systems
- security guards
- access cards

Even if one protection fails, others remain.

Cybersecurity uses the same principle.

## Examples of Security Layers

Common layers used in organizations include:

Network security:
- firewalls
- network monitoring

Account security:
- strong passwords
- multi-factor authentication

System security:
- operating system updates
- secure configurations

Monitoring systems:
- logs
- security alerts

Each layer contributes to stronger protection.

## Defense-in-Depth in Practice

Layers work together.

For example, an attacker may try to log into a company system.

Defense layers could include:
- password protection
- multi-factor authentication
- login monitoring
- alerts for unusual activity

Even if a password is stolen, other protections may still block the attacker.

## Why This Concept Is Important

Defense-in-Depth helps organizations:
- reduce security risks
- detect attacks earlier
- limit damage if a breach occurs

Blue Teams design security systems so that multiple protections work together.

## Key Takeaways

- Defense-in-Depth uses multiple layers of security.
- No single security control is enough on its own.
- Different protections work together to reduce risk.
- Layered defenses help organizations detect and stop attacks.
